# 模块6：用户深耕

## 6.1 如何提升新用户留存（上）

### 新用户留存的重要性

- 获取新用户成本高（通常是老用户的5-10倍）
- 留存决定了LTV（用户生命周期价值）
- 是产品价值是否被用户认可的验证

### 魔法数字

**定义**：用户发生特定行为N次后的留存显著提升，这个N就是魔法数字

**关键图表：魔法数字数据表**

| 点击喜欢次数 | 次留率 | 边际效用 |
|--------------|--------|----------|
| 0次 | 21% | - |
| 1次 | 34.62% | +13.62% |
| 2次 | 44.93% | +10.31% |
| 3次 | 64.78% | +19.85% |
| 4次 | 74.49% | +9.71% |
| 5次 | 75.74% | +1.25% |
| 6次 | 77.44% | +1.70% |
| 7次 | 85.43% | +7.99% |
| 8次 | 91.58% | +6.15% |
| 9次 | 91.67% | +0.09% |
| 10次 | 92.59% | +0.92% |

**关键洞察**：
- 0次到3次之间边际效用最大（+43.78%）
- 8次以后边际效用趋近于0
- **魔法数字约8次**：用户2天内点击喜欢歌曲8次，留存率达91.58%

### 激活行为四步法

| 步骤 | 说明 | 操作 |
|------|------|------|
| 1. 找出时间窗口 | 用户行为发生在多长时间内 | 分析数据确定观察周期 |
| 2. 找出激活行为 | 什么行为与留存正相关 | 相关性分析 |
| 3. 找出最佳次数 | 边际效用最大的次数 | 分组测试 |
| 4. 验证因果性 | 通过A/B测试验证 | 实验设计 |

## 6.2 如何提升新用户留存（下）

### 新用户引导的场景与触点

| 场景 | 触点 | 常用形式 |
|------|------|----------|
| 用户首次使用产品时 | 首次启动引导页 | 步骤引导、浮层提示 |
| 在用户的任务进程中 | 网站/App页面 | 进度条、教程提示 |
| 当用户遇到问题时 | 帮助中心、弹窗 | 客服、FAQ |
| 希望用户发生某行为时 | Push、短信 | 召回通知 |

### 常用引导形式

**关键图表**：新用户引导形式对比

| 形式 | 特点 | 适用场景 |
|------|------|----------|
| 步骤引导 | 结构化、强制性 | 复杂任务 |
| 浮层提示 | 轻量、上下文相关 | 功能解释 |
| 视频教程 | 直观、详细 | 复杂操作 |
| 渐进披露 | 简洁、按需 | 简单任务 |

### 成功引导的相关因素

**关键图表**：成功引导的五个关键因素

| 因素 | 说明 |
|------|------|
| 时机恰当 | 不过早不迟到，在用户需要时出现 |
| 内容简洁 | 聚焦核心，不给用户造成负担 |
| 操作便捷 | 步骤少，一键完成 |
| 反馈清晰 | 用户知道操作结果 |
| 个性化 | 根据用户特征定制引导 |

## 6.3 如何提升老用户留存（上）

### 老用户流失原因

| 类型 | 说明 | 示例 |
|------|------|------|
| 需求消亡 | 用户不再需要该产品 | 产品过时、需求转移 |
| 兴趣降低 | 对产品失去新鲜感 | 疲劳感、新鲜感消退 |
| 被动流失 | 外部因素导致 | 体验差、被竞品吸引 |
| 资源消耗 | 产品提供的价值用尽 | 权益用尽、积分耗尽 |
| 权益被侵犯 | 用户感到不公平 | 规则变化、差异化待遇 |

### 老用户留存策略

**增强动力**：

| 策略 | 说明 |
|------|------|
| 提升用户体验 | 让用户持续感到价值 |
| 建设激励体系 | 积分、等级、勋章 |
| 促进用户活跃 | 签到、任务、内容更新 |

**增大阻力**：

| 策略 | 说明 |
|------|------|
| 培养用户习惯 | 使用频率稳定 |
| 增大迁移成本 | 关系链、数据、习惯 |

## 6.4 如何提升老用户留存（下）

### 用户体验的定义

用户体验（User Experience，UE/UX）指的是用户亲身参与的一种经历以及在整个过程中获得的全部感受。

**核心观点**：
> 在所有增长运营的策略中，用户体验是最重要的基石——无论是用户留存，还是用户的价值产出，都植根于良好的用户体验。

### 体验是如何产生的

| 类型 | 说明 |
|------|------|
| 使用前体验 | 预期、吸引、兴趣 |
| 使用中体验 | 操作、效率、满足 |
| 使用后体验 | 回忆、评价、推荐 |
| 直接体验 | 用户自身使用产品 |
| 间接体验 | 用户通过他人了解产品 |

### 用户体验评估五度模型

**关键图表：五度模型结构**

| 标准 | 含义 |
|------|------|
| **吸引度** | 吸引目标用户的注意力，激发其兴趣，唤起其欲望（包含AIDA四个完整过程） |
| **完成度** | 用户通过产品来解决问题时，实际解决的效率、成功率、完成率等 |
| **满意度** | 用户在体验实际产品后的看法或感觉，是否满意、愉悦、惊喜等 |
| **忠诚度** | 用户在首次体验后，是否还会回访？如再次回到产品中，复购、黏性等 |
| **推荐度** | 用户是否愿意自发地向朋友推荐、传播良好口碑等 |

### 如何打造好的用户体验

| 方法 | 说明 | 示例 |
|------|------|------|
| **核心价值** | 深化产品能给用户的价值 | 找到美、放大美、丰富美、塑造美 |
| **UCD理念** | 以用户为中心的设计 | ISO 9241-210标准方法 |
| **触点体验** | 基于用户体验地图优化触点 | 京东数坊案例五步法 |
| **峰终体验** | 关键时刻的峰值体验 | 峰值和结束时的感受 |
| **个性化运营** | 因人而异的精准服务 | 推荐、定制服务 |

### UCD理念（以用户为中心的设计）

**ISO 9241-210 标准要求**：
- 根据对用户、任务和环境的清晰认识进行设计
- 用户参与设计和研发的全过程
- 设计是由以用户为中心的评估推动和改进的
- 流程是反复的
- 设计体现了全部用户体验
- 设计团队包括多学科技术和视角

### 触点体验优化：京东数坊案例

**项目背景**：京东DMP下的人群定投工具，希望识别用户体验改善和产品增长机会。

**分析五步法**：

| 步骤 | 操作 | 产出 |
|------|------|------|
| 1. 明确目标 | 明确用户需求是快速成功生成人群包 | 确定分析方向 |
| 2. 梳理用户旅程 | 选择人群→设置规则→生成人群包 | 识别关键阶段 |
| 3. 触点映射 | 识别各阶段的触点位置和用户情绪 | 发现体验痛点 |
| 4. 分析痛点 | 人群生成速度及成功率不理想 | 明确优化方向 |
| 5. 应用优化 | 基于策略环方法进行应用 | 落地改进 |

## 6.5 应对处理用户流失的技巧

### 用户召回策略

| 策略 | 适用场景 | 核心动作 |
|------|----------|----------|
| Push召回 | App用户活跃 | 推送通知激活 |
| 短信召回 | 有手机号 | 短信触达提醒 |
| 邮件召回 | 有邮箱 | 邮件营销唤醒 |
| 活动召回 | 有活动预算 | 大促活动吸引 |
| 专属福利 | 高价值用户 | VIP专属优惠 |

### 召回话术要点

- **唤起美好回忆**：提及用户过去的美好体验
- **突出变更改进**：说明产品已优化改进
- **给出回归理由**：限时优惠、专属福利

## 6.6 持续深耕用户LTV

### LTV提升路径

```
提升复购率 → 提升客单价 → 提升推荐率
```

### LTV深耕策略

| 策略 | 说明 |
|------|------|
| 会员体系 | 等级权益激励复购 |
| 积分体系 | 行为积分兑换奖励 |
| 专属服务 | VIP专属体验 |
| 关联推荐 | 提升客单价 |

---

## 本模块要点

1. **魔法数字**：用户激活需要达到一定次数，新用户激活率才能显著提升
2. **新用户引导**：时机、内容、操作、反馈、个性化是关键
3. **用户体验五度模型**：吸引度→完成度→满意度→忠诚度→推荐度
4. **老用户留存**：增强动力（体验、激励、活跃）+ 增大阻力（习惯、迁移成本）
5. **LTV提升**：复购率→客单价→推荐率的递进路径
