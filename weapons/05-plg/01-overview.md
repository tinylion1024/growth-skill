# 产品驱动增长武器库（PLG）

> 让产品成为增长引擎的15个武器

---

## 概述

产品驱动增长（PLG）是通过产品本身实现增长。核心是让产品价值成为最主要的获客和留存因素。

```
PLG核心公式：

Product Growth = Activation × Retention × Referral

核心逻辑：
好产品 → 用户自发使用 → 发现价值 → 自发传播
```

---

## 武器清单（15个）

### 51. Freemium模式

```
战术：免费基础功能+付费高级功能

设计原则：
- 免费功能要能解决核心问题
- 付费功能要有明显差异
- 升级路径要清晰

免费层设计：
- 核心功能限使用量
- 功能子集
- 协作限制

付费层设计：
- 无限制使用
- 高级功能
- 团队协作
- 优先支持

案例：
- Slack：免费有限消息
- Notion：免费有限block
- Canva：免费有限模板

关键指标：
- 免费→付费转化率
- 付费用户LTV
```

### 52. 免费试用

```
战术：让用户体验完整产品后再决定

试用设计：
- 全功能试用
- 时间限制（7/14/30天）
- 需要信用卡（增加转化）
- 自动升级（避免流失）

优化要点：
- 试用开始要简单
- 快速让用户看到价值
- 试用结束前要提醒
- 升级路径要清晰

案例：
- Dropbox：免费试用
- Adobe Creative Cloud：7天试用
- HubSpot：免费CRM

关键指标：
- 试用率
- 试用→付费转化率
- 试用流失原因
```

### 53. 无需登录体验

```
战术：让用户先体验，再要求注册

设计：
- 打开产品即可使用核心功能
- 体验一定次数/时间后要求注册
- 注册过程要简化

适用场景：
- 工具型产品
- 内容型产品
- 游戏

案例：
- Figma：无需登录即可编辑
- Canva：无登录可使用模板
- Office Online：在线使用

关键点：
- 未登录体验要完整
- 注册时说明利益
- 体验数据要保留
```

### 54. 快速Onboarding引导

```
战术：快速让用户感受到产品价值

Onboarding设计：
1. 欢迎+价值承诺
2. 3步快速上手
3. 第一个成果展示
4. 成就感庆祝

关键要素：
- 不要让用户迷茫
- 每步都有反馈
- 快速到"aha moment"
- 可以跳过

案例：
- Notion：3步创建第一个workspace
- Slack：邀请团队快速聊天
- Linear：导入项目快速上手

关键指标：
- Onboarding完成率
- 激活率
- Time to Value
```

### 55. 产品内引导（Tooltip）

```
战术：用工具提示引导用户使用

引导类型：
- 功能介绍
- 操作提示
- 快捷键提示
- 最佳实践

设计原则：
- 轻柔不打扰
- 上下文相关
- 可以关闭
- 不要重复

案例：
- Figma的工具提示
- Notion的内联提示
- Linear的快捷键提示

关键点：
- 看到问题时才出现
- 简洁明了
- 有链接可以深入了解
```

### 56. 模板库

```
战术：用模板降低用户使用门槛

模板类型：
- 使用模板
- 内容模板
- 设计模板
- 文档模板

增长逻辑：
模板 → 用户使用 → 发现价值 → 持续使用

案例：
- Notion：模板库
- Canva：模板市场
- Figma：社区模板

关键点：
- 模板质量要高
- 分类要清晰
- 搜索要方便
- 更新要及时
```

### 57. 空状态设计（Empty state）

```
战术：在用户没有任何数据时引导上手

空状态要素：
- 说明当前状态
- 说明为什么重要
- 展示示例/案例
- 明确的行动引导

设计要点：
- 视觉要吸引
- 说明要有价值
- 行动要简单
- 示例要真实

案例：
- "还没有文档，创建第一个吧"
- "邀请团队成员，开始协作"
- "上传文件，开始整理"

关键点：
- 空状态是教育用户的机会
- 不要只是"空"，要有引导
- 可以根据用户类型定制
```

### 58. 产品内分享按钮

```
战术：在产品使用过程中设计分享触点

分享场景：
- 完成某个操作后
- 创建内容后
- 获得成果后

分享内容：
- 作品成果
- 统计/报告
- 邀请链接

设计要点：
- 分享要自然
- 分享内容要有价值
- 分享后要有回报

案例：
- Figma分享按钮
- Notion分享发布
- Canva一键分享
```

### 59. 产品水印传播

```
战术：在用户创建的内容上留下品牌水印

适用场景：
- 设计工具
- 内容创作
- 图片编辑

设计要点：
- 水印不要太明显
- 要有专业的Look
- 可以提供去水印付费

案例：
- Canva：部分模板带水印
- Figma：分享时显示Logo

关键点：
- 不要让用户反感
- 水印是品牌曝光
- 可以设计精美的水印
```

### 60. 团队协作功能

```
战术：通过团队使用实现用户增长

设计：
- 免费创建团队
- 邀请协作者
- 协作功能促进传播

增长逻辑：
个人使用 → 发现协作价值 → 创建团队 → 邀请同事 → 团队增长

案例：
- Slack：工作空间邀请
- Notion：团队工作区
- Figma：团队文件库

关键点：
- 协作功能要真的好
- 邀请要方便
- 体验要流畅
```

### 61. 使用量限制触发

```
战术：在用户快达到限制时触发付费转化

限制类型：
- 文件数量
- 存储空间
- API调用
- 功能使用

触发设计：
- 达到80%时提醒
- 达到100%时提示升级
- 提供升级方案

案例：
- GitHub：私有仓库限制
- Vercel：带宽限制
- OpenAI：Token限制

关键点：
- 限制要合理
- 提醒要温和
- 升级要方便
- 价值要明确
```

### 62. 成果导出触发传播

```
战术：用户导出成果时引导分享

导出场景：
- 导出设计稿
- 导出报告
- 导出数据

设计：
- 导出时提示分享
- 导出的文件包含品牌信息
- 提供"分享到X"的选项

案例：
- 设计工具导出时提示分享
- 报告工具导出PDF带Logo

关键点：
- 导出是用户有成就感的时刻
- 分享要自然
- 品牌露出要温和
```

### 63. 使用报告自动发送

```
战术：定期向用户发送使用报告

报告内容：
- 使用统计
- 成果展示
- 活跃度分析
- 价值回顾

发送频率：
- 每周/每月

增长逻辑：
使用报告 → 用户感受到价值 → 持续使用 → 分享传播

案例：
- Spotify Wrapped
- GitHub Contributions
- Notion Stats

关键点：
- 报告要有价值
- 设计要好看
- 要有分享元素
- 要引导下一步
```

### 64. 内置社区入口

```
战术：在产品内嵌入社区入口

入口设计：
- 帮助中心
- 社区论坛
- 模板市场
- 反馈渠道

案例：
- Figma：内置Community
- Notion：内置Templates/Gallery
- Linear：内置Changelog

关键点：
- 入口要自然
- 内容要相关
- 要有互动机会
```

### 65. 产品内推荐系统

```
战术：在产品内推荐相关功能/内容

推荐类型：
- 功能推荐（新功能提示）
- 内容推荐（相关模板）
- 用户推荐（可能感兴趣的人）
- 资源推荐（学习资料）

案例：
- Notion：建议使用模板
- Figma：建议使用组件
- Linear：建议关联Issue

关键点：
- 推荐要相关
- 不要过于打扰
- 要有明确的价值
```

## PLG检查清单

```
PLG效果评估：

□ 用户能快速感受到价值？
□ 免费→付费转化率如何？
□ 产品内是否有自然传播触点？
□ 团队协作是否带来增长？

关键指标：
- 激活率（完成关键行为）
- 留存率（持续使用）
- 付费率（转化付费）
- 传播率（邀请新用户）
```
