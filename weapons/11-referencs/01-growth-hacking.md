# 增长黑客派

> Sean Ellis | 数据驱动的实验增长方法论

---

## 核心理论

**增长黑客 = 用最小成本获取最大增长**

核心公式：`Growth = f(Experiment, Data, Speed)`

核心原则：
1. **数据驱动决策** — 一切基于数据，而非直觉
2. **快速实验迭代** — 小步快跑，快速验证
3. **低成本高回报** — 精益思维，避免浪费
4. **全员参与增长** — 增长是每个人的责任

---

## 关键框架

### A/B测试流程

```
假设建立 → 设计实验 → 分配流量 → 收集数据 → 分析结果 → 迭代优化

关键点：
- 每次只改一个变量
- 保证样本量统计显著
- 记录所有实验结果
```

### ICE 优先级框架

```
ICE = Interest × Confidence / Ease

Interest（兴趣）：用户对此有多重要？
Confidence（信心）：我对结果有多确定？
Ease（易度）：实现需要多少资源？

优先做 ICE 分数最高的实验
```

### 增长漏斗

```
Awareness → Acquisition → Activation → Retention → Revenue → Referral
   流量       注册           激活        留存        变现        推荐

优化优先级（从易到难）：
1. 转化率优化（每一步的%）
2. 流失点识别（哪里流失最多）
3. 漏斗宽度（每个阶段用户量）
```

## 常见误区

```
❌ 迷信A/B测试 - A/B测试是工具，不是答案
❌ 只看转化率 - 转化率高不等于增长
❌ 忽视长期价值 - 追求短期指标，牺牲用户体验
❌ 盲目实验 - 没有假设的实验是浪费

✅ 正确做法：
- 用数据驱动，但用洞察引导
- 关注全漏斗，不只是转化
- 平衡短期和长期
- 实验要有假设支撑
```
