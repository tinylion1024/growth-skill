# 病毒传播

> 让用户主动传播 = 最低成本的指数级增长

---

## 核心理论

**病毒传播 = 让用户成为传播者，让分享成为产品体验的一部分**

```
病毒传播公式：
K = i × q

K = 病毒系数（每个用户带来的新用户数）
i = 传播率（每个用户发送的邀请数）
q = 转化率（收到邀请的人转化为新用户的比例）

关键门槛：K > 1

K < 1：线性增长或负增长
K = 1：持平（增长停滞）
K > 1：指数级增长 ✓

案例：Dropbox
- 早期K值约1.5（每个用户带来1.5个新用户）
- 3年内从10万用户增长到4000万用户
```

## 关键框架

### 病毒传播设计要素

```
┌─────────────────────────────────────────────────────┐
│                                                     │
│  触发分享的时刻                                     │
│      │                                             │
│      ▼                                             │
│  用户获得价值 ──▶ 分享动机 ──▶ 分享渠道              │
│      │                │              │             │
│      ▼                ▼              ▼             │
│  产品体验         激励设计        分享体验          │
│  足够好           双向奖励        低摩擦            │
│                                                     │
│  收到邀请                                       │
│      │                                           │
│      ▼                                           │
│  新用户体验 ──▶ 价值感知 ──▶ 成为传播者             │
│                                                     │
└─────────────────────────────────────────────────────┘
```

### 分享动机设计

```
强分享动机（高传播率）：
- 经济利益（双方得奖励）
- 社交认同（分享成就/位置）
- 产品体验（好东西想分享）
- 社会议题（价值观表达）

弱分享动机（低传播率）：
- 完成任务（功利性强）
- 单边利益（只有邀请人得益）
- 高摩擦（需要太多步骤）
```

### 分享摩擦消除

```
分享摩擦来源：
1. 需要登录/注册
2. 分享步骤太多
3. 找不到分享入口
4. 分享内容不吸引人
5. 奖励不明显

解决方案：
- 一键分享
- 默认分享
- 分享即奖励
- 让分享成为体验的一部分
```

## 病毒传播检测清单

```
□ 是否有明确的分享触发点？
□ 分享是否足够简单（一键/默认）？
□ 是否有双向激励（邀请人+被邀请人）？
□ 分享内容是否有吸引力（模板化/个性化）？
□ 邀请到达率如何（邮件/短信/社交）？
□ 被邀请人转化率如何（q值）？
□ 整体K值是否 > 1？
□ 病毒传播是否可持续（非一次性）？
```
