# 用户画像体系

## 两种用户画像对比

| 类型 | 英文 | 性质 | 制作方式 | 适用场景 |
|------|------|------|----------|----------|
| 用户角色 | User Persona | 虚拟人物 | 定性研究 | 产品开发、团队认知 |
| 用户档案 | User Profile | 真实数据 | 定量计算 | 精准投放、精细运营 |

---

## User Persona（用户角色）

### 定义

用户角色是一个**虚构的人物角色**，用来代表可能以类似方式使用网站、品牌或产品的用户类型。

### 关键信息

| 信息类型 | 说明 | 示例 |
|----------|------|------|
| 基本特征 | 人口统计特征 | 姓名、年龄、性别、收入 |
| 用户需求 | 想要什么 | "随时随地观看喜欢的电影" |
| 用户行为 | 做什么 | "热衷于多屏观看" |
| 用户心象 | 为什么这么想 | 认知、动机、情感、价值观 |

### 创建步骤

1. **明确目的、选择人群**：确定目标用户群体
2. **收集数据、研究用户**：通过访谈、调研收集信息
3. **提炼特征、创建画像**：提取共性特征，构建虚拟人物
4. **画像验证、优化完善**：验证画像准确性，持续优化

---

## User Profile（用户档案）

### 定义

用户档案是基于用户产生的**实际信息**来画像，是根据用户属性、用户行为等数据，提取、计算出的**标签化的用户特征集合**。

### 标签分类

**按业务角度分**：
- 属性标签：性别、年龄、地区
- 行为标签：浏览、收藏、购买
- 价值标签：高价值、中价值、低价值
- 偏好标签：品牌偏好、价格敏感度

**按技术角度分**：

| 标签类型 | 说明 | 示例 |
|----------|------|------|
| 事实标签 | 客观事实描述 | 性别（男/女/未知） |
| 规则标签 | 满足规则计算得出 | RFM价值（重要价值/重要保持/...） |
| 预测标签 | 基于数据预测未来 | 流失预测（预流失/正常） |

### RFM价值模型

| 类型 | 说明 | 特征 |
|------|------|------|
| 重要价值 | 高频高消费 | 优质客户，重点维护 |
| 重要保持 | 长期未消费但曾高消费 | 需召回激活 |
| 重要发展 | 低频但高消费潜力 | 重点发展 |
| 重要挽留 | 即将流失的高价值 | 重点挽留 |
| 一般价值/保持/发展/挽留 | 其他分层 | 常规运营 |

## 应用场景

| 场景 | User Persona | User Profile |
|------|--------------|--------------|
| 产品开发 | ✅ 理解用户需求 | ❌ |
| 团队认知统一 | ✅ 建立共同语言 | ❌ |
| 用户获取 | ❌ | ✅ LookAlike相似人群 |
| 精细化运营 | ❌ | ✅ 差异化运营策略 |
| 精准投放 | ❌ | ✅ 定向投放 |
| 搜索优化 | ❌ | ✅ 关键词优化 |
| 风险控制 | ❌ | ✅ 异常用户识别 |

## 用户标签系统

### 标签维度

```
用户基础属性 → 用户行为 → 用户价值 → 用户偏好
```

### 标签示例

| 标签名称 | 标签值 | 标签产生方式 | 标签类别 |
|----------|--------|--------------|----------|
| 性别 | 男/女/未知 | 系统标注 | 事实标签 |
| 移动操作系统 | Android/iOS/鸿蒙 | 系统标注 | 事实标签 |
| 是否关注公众号 | 是/否 | 系统标注 | 事实标签 |
| RFM价值 | 重要价值/重要保持/... | 根据RFM计算 | 规则标签 |
| 流失预测 | 预流失/正常 | 模型计算 | 预测标签 |

## 用户画像应用案例

### 京东数坊案例

**项目背景**：京东DMP下的人群定投工具，希望识别用户体验改善和产品增长机会。

**分析过程**：
1. 明确目标：用户需求是快速成功生成人群包
2. 梳理用户旅程：选择人群→设置规则→生成人群包
3. 触点映射：识别各阶段的用户体验痛点
4. 分析痛点：人群生成速度及成功率不理想
5. 应用优化：基于策略环方法进行优化

---

## 小结

- User Persona用于产品设计和团队认知
- User Profile用于精细化运营和精准投放
- 标签体系是用户运营的基础设施
- RFM模型是价值分层的经典方法
