## 核心理念

```
┌─────────────────────────────────────────────────────────────────┐
│                                                                 │
│   增长 ≠ 赚钱                                                   │
│                                                                 │
│   传统思维：增长优先         商业思维：增长必须健康               │
│   ─────────────            ──────────────                      │
│   用户增长 = 成功           用户增长 + 盈利 = 成功                │
│   规模越大越好              规模要匹配商业模式                    │
│   后期考虑盈利              盈利是增长的前提                     │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘

核心原则：
1. **LTV > CAC** - 用户必须赚回成本
2. **留存决定命运** - 没有留存，增长是空谈
3. **指标驱动决策** - 数据验证商业模型
```

### 增长健康度判断

```
增长健康度矩阵：

                    LTV/CAC > 3        LTV/CAC < 3
                    ─────────────      ─────────────
  流失率 < 5%/月    ██ 健康发展        ██ 优化效率
  月流失率

  流失率 > 10%/月   ██ 加速获客        ██ 危险信号

建议：
- LTV/CAC > 3 + 流失 < 5%：加速增长
- LTV/CAC < 3 + 流失 < 5%：优化获客效率
- LTV/CAC > 3 + 流失 > 10%：专注留存
- LTV/CAC < 3 + 流失 > 10%：停止增长，先解决问题
```

## 商业模型类型

### SaaS 商业模型

```
SaaS 收入模式：

1. 订阅模式 (Subscription)
   - 月/年订阅
   - 稳定收入
   - 例：Slack、Notion

2. 按使用量付费 (Usage-based)
   - 基于使用量收费
   - 收入与价值挂钩
   - 例：AWS、Twilio

3. 混合模式 (Hybrid)
   - 基础订阅 + 超额使用费
   - 稳定 + 弹性
   - 例：GitHub、Figma

收入确认：
- Monthly Recurring Revenue (MRR)
- Annual Recurring Revenue (ARR)
- Net New ARR（新增ARR）
- Expansion ARR（扩张ARR）
```

### Marketplace 商业模型

```
Marketplace 收入模式：

1. 交易佣金
   - 每次交易收取百分比
   - 最常见模式
   - 例：Uber、Airbnb

2. 会员费
   - 卖家支付月费/年费
   - 收入稳定
   - 例：eBay订阅

3. 广告费
   - 搜索排名广告
   - 高利润
   - 例：亚马逊赞助广告

4. 增值服务
   - 物流、支付等服务
   - 高毛利
   - 例：天猫国际

Marketplace 关键指标：
- GMV（平台总交易额）
- Take Rate（佣金率）
- 买家/卖家比
- 留存率
```

### 电商商业模型

```
电商收入模式：

1. 零售价差
   - 采购 → 销售
   - 赚取差价
   - 例：京东

2. 平台佣金
   - 平台模式
   - 收取佣金
   - 例：天猫

3. 订阅电商
   - 定期配送
   - 稳定现金流
   - 例：Dollar Shave Club

4. 会员制
   - 会员费换取折扣
   - 高忠诚度
   - 例：亚马逊Prime

电商关键指标：
- GMV（成交总额）
- 订单数（AOV）
- 毛利率
- 复购率
- 客户获取成本（CAC）
```

## 商业模型验证

### 精益分析框架

```
精益分析四步法：

Step 1: 找到那个关键指标（One Metric That Matters）
   - 现阶段最重要的指标是什么？
   - 这个指标能反映核心假设？

Step 2: 建立基准线
   - 当前指标值是多少？
   - 与行业比如何？
   - 与目标比如何？

Step 3: 设定目标
   - 目标值是多少？
   - 什么时候达成？
   - 如何衡量？

Step 4: 设计实验
   - 什么实验能改善这个指标？
   - 优先级是什么？
   - 如何验证？
```

### 单位经济验证

```
单位经济验证框架：

┌─────────────────────────────────────────────────────────────────┐
│                                                                 │
│   基础公式：                                                     │
│                                                                 │
│   LTV = ARPU × 留存月数 × 毛利率                                │
│                                                                 │
│   验证标准：                                                     │
│   - LTV/CAC > 3                                                 │
│   - Payback Period < 12个月                                     │
│   - 流失率 < 5%/月（SaaS）                                     │
│                                                                 │
│   如果不达标，问题在哪？                                         │
│                                                                 │
│   ┌─────────────────────────────────────────────────────────┐   │
│   │                                                         │   │
│   │   问题        可能原因           解决方案                │   │
│   │   ────────   ─────────────      ─────────              │   │
│   │   LTV太低    定价太低           提高定价                 │   │
│   │              用户生命周期短      提升留存               │   │
│   │                                                         │   │
│   │   CAC太高    渠道效率低         优化渠道                 │   │
│   │              转化率低            优化漏斗                 │   │
│   │                                                         │   │
│   │   流失高     产品价值不足        提升产品                 │   │
│   │              用户匹配不对        优化获客                 │   │
│   │                                                         │   │
│   └─────────────────────────────────────────────────────────┘   │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```
