# 增长飞轮 - 病毒引擎

> K-Factor > 1 | 让用户主动传播

---

## 病毒引擎原理

```
病毒传播模型：

┌─────────────────────────────────────────────────────────────────┐
│                                                                 │
│   用户A ──邀请──▶ 用户B ──邀请──▶ 用户C ──邀请──▶ 用户D         │
│      │                  │                  │                     │
│      ▼                  ▼                  ▼                     │
│   分享链接           分享链接           分享链接                  │
│      │                  │                  │                     │
│      │                  │                  │                     │
│   转化率c            转化率c            转化率c                  │
│      │                  │                  │                     │
│      ▼                  ▼                  ▼                     │
│   新用户            新用户             新用户                    │
│                                                                 │
│   K = i × c                                                         │
│   i = 每个用户发送的邀请数                                         │
│   c = 每个邀请的转化率                                             │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘

病毒系数判断：
- K > 1：病毒式增长（每个用户带来超过1个新用户）
- K < 1：衰减式增长（最终会停止）
- K = 1：平衡点（增长停滞）
```

---

## 病毒传播类型

```
病毒传播分类：

1. 原生病毒（Native Viral）
   - 用户为了获得产品价值而主动传播
   - 例：Hotmail的"Get Your Free Email"底部署名

2. 激励病毒（Incentivized Viral）
   - 用户因奖励而传播
   - 例：Dropbox的"邀请朋友获得空间"

3. 口碑病毒（Word of Mouth）
   - 用户因喜欢产品而推荐
   - 例：Uber的好体验带来的推荐

4. 场景病毒（Contextual Viral）
   - 用户在特定场景下分享
   - 例：抖音的"拍视频分享"

原生病毒 > 激励病毒 > 口碑病毒 > 场景病毒
（成本低、转化高）    （成本高、易衰减）
```

### 病毒循环设计

```
病毒循环四要素：

┌─────────────────────────────────────────────────────────────────┐
│                                                                 │
│                    触发 (Trigger)                               │
│                       │                                         │
│                       ▼                                         │
│   ┌─────────────────────────────────────────────────────────┐   │
│   │                                                         │   │
│   │   用户收到外部触发（通知、邮件等）                        │   │
│   │         或内部触发（情绪、习惯）                          │   │
│   │                                                         │   │
│   └─────────────────────────────────────────────────────────┘   │
│                       │                                         │
│                       ▼                                         │
│   ┌─────────────────────────────────────────────────────────┐   │
│   │                                                         │   │
│   │              行动 (Action)                               │   │
│   │                                                         │   │
│   │   用户完成简单的分享/邀请动作                             │   │
│   │   关键：动作要简单、摩擦要低                              │   │
│   │                                                         │   │
│   └─────────────────────────────────────────────────────────┘   │
│                       │                                         │
│                       ▼                                         │
│   ┌─────────────────────────────────────────────────────────┐   │
│   │                                                         │   │
│   │              奖励 (Reward)                               │   │
│   │                                                         │   │
│   │   用户和被邀请人都获得奖励                               │   │
│   │   关键：奖励要即时、有感知                               │   │
│   │                                                         │   │
│   └─────────────────────────────────────────────────────────┘   │
│                       │                                         │
│                       ▼                                         │

│   │                                                         │   │
│   │              投入 (Investment)                           │   │
│   │                                                         │   │
│   │   被邀请人使用产品、积累价值                             │   │
│   │   关键：价值积累让用户留存                               │   │
│   │                                                         │   │
│   └─────────────────────────────────────────────────────────┘   │
│                       │                                         │
│                       └──────────────────────────────▶         │
│                                               新用户成为分享者  │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### 提高K值的策略

```
提高K值的两种方式：

1. 提高邀请数（i）
   ┌─────────────────────────────────────────────────────────┐
   │                                                         │
   │   设计更好的邀请触发点：                                 │
   │   - 用户完成关键行为后邀请                               │
   │   - 用户达到里程碑后邀请                                 │
   │   - 用户与朋友互动场景下邀请                             │
   │                                                         │
   │   降低邀请摩擦：                                         │
   │   - 一键分享                                             │
   │   - 预设分享文案                                         │
   │   - 多种分享渠道                                         │
   │                                                         │
   └─────────────────────────────────────────────────────────┘

2. 提高邀请转化率（c）
   ┌─────────────────────────────────────────────────────────┐
   │                                                         │
   │   优化邀请体验：                                         │
   │   - 邀请内容与用户相关                                   │
   │   - 被邀请人能看到邀请人信息                             │
   │   - 邀请价值主张清晰                                     │
   │                                                         │
   │   优化落地页：                                           │
   │   - 承诺立即体验                                         │
   │   - 减少注册步骤                                         │
   │   - 快速展示核心价值                                     │
   │                                                         │
   └─────────────────────────────────────────────────────────┘
```

## 经典病毒案例

### Dropbox

```
Dropbox病毒飞轮：

Step 1: 用户完成注册
   └─ 注册后获得16GB免费空间

Step 2: 用户投入内容
   └─ 用户上传文件，积累数据

Step 3: 分享触发
   └─ 用户想获得更多空间 → 邀请朋友

Step 4: 邀请转化
   └─ 朋友通过链接注册 → 双方获得空间

Step 5: 新用户留存
   └─ 新用户上传文件 → 循环继续

K-Factor = 0.35（每个用户邀请0.35个新用户）
虽然K < 1，但成本极低（边际成本 ≈ 0）
```

### Hotmail

```
Hotmail原生病毒：

背景：
   - 1996年，Hotmail是第一个免费网页邮箱
   - 当时的邮件客户端都需要安装软件

病毒设计：
   - 所有通过Hotmail发出的邮件底部自动加上
   - "Get your free email at Hotmail"

效果：
   - 6个月内获得1200万用户
   - 18个月内达到8500万用户
   - K-Factor极高（每个用户自然传播给多人）

关键洞察：
   - 分享行为自然发生（发邮件时自动分享）
   - 无需激励，用户无感知
   - 触达的是真实社交关系
```

### Referral Links（推荐链接）

```
推荐链接进化：

1.0 简单推荐链接
   - 分享个人链接
   - 手动邀请

2.0 分享渠道优化
   - 一键分享到社交媒体
   - 预设分享文案

3.0 实时奖励
   - 邀请人和被邀请人同时获得奖励
   - 奖励即时到账

4.0 社交证明
   - 显示"你的朋友XXX也在用"
   - 朋友头像显示
   - 实时动态通知

5.0 游戏化
   - 邀请排行榜
   - 阶梯奖励
   - 成就徽章
```

## 病毒引擎指标

```
病毒引擎核心指标：

1. K-Factor（病毒系数）
   - 计算：K = i × c
   - 目标：K > 1

2. 邀请率（Invite Rate）
   - 定义：每用户发送邀请数
   - 计算：总邀请数 / 总用户数

3. 转化率（Conversion Rate）
   - 定义：每邀请产生的新用户数
   - 计算：总新用户 / 总邀请数

4. 病毒周期（Viral Cycle Time）
   - 定义：从邀请到转化的平均时间
   - 目标：越短越好

病毒效率计算：
病毒贡献率 = 病毒获取用户 / 总新增用户

例：本周新增1000用户，其中400来自邀请
病毒贡献率 = 400/1000 = 40%
```

## 病毒引擎优缺点

```
优点：
- 获客成本极低（接近0）
- 用户质量高（朋友推荐）
- 增长非线性
- 自驱动增长

缺点：
- 难以控制和预测
- 依赖产品/市场匹配
- 可能被滥用（作弊）
- 不稳定（可能突然停止）

适用场景：
- 产品有强烈价值主张
- 用户有分享动机
- 分享行为自然发生

不适用场景：
- 产品价值难以通过文字传达
- 用户没有分享场景
- 获客周期长（B2B）
```
