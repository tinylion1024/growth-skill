# 增长飞轮 - 粘性引擎

> 留存 > 流失 | 让用户持续回来

---

## 粘性引擎原理

```
粘性增长公式：

 Retention > Churn

只要留存率 > 流失率
用户池就会自然增长

┌─────────────────────────────────────────────────────────────────┐
│                                                                 │
│   用户池                                                       │
│   ┌─────────────────────────────────────────────────────────┐   │
│   │                                                         │   │
│   │   新增用户 ──────────────▶ 用户池                       │   │
│   │        │                                             │   │
│   │        │      流失率 < 留存率                         │   │
│   │        │      = 净增长                                │   │
│   │        │                                             │   │
│   │   ◀────────────── 流失用户                            │   │
│   │                                                         │   │
│   └─────────────────────────────────────────────────────────┘   │
│                                                                 │
│   关键：让用户持续获得价值，形成使用习惯                         │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

## 粘性机制设计

### 四种粘性来源

```
粘性四象限：

                    用户依赖度
                        ▲
                        │
              社交关系   │    产品功能
            ┌────────────┼────────────┐
            │            │            │
 高         │  社交锁定   │   功能锁定   │
            │            │            │
            │            │            │
            │            │            │
 低         │  奖励锁定   │   习惯锁定   │
            │            │            │
            │            │            │
            └────────────┼────────────┘
                        │
                        ▼
                   用户主动意愿

1. 社交关系（Social）
   - 朋友在这里，你不能离开
   - 例：微信、WhatsApp

2. 产品功能（Functional）
   - 产品功能不可替代
   - 例：Photoshop、Excel

3. 奖励锁定（Reward）
   - 积累的奖励让用户不愿放弃
   - 例：蚂蚁森林、积分体系

4. 习惯锁定（Habit）
   - 使用习惯让用户自然回来
   - 例：Duolingo、Keep
```

## 留存曲线分析

### 三种留存曲线

```
1. 快乐台车型（The Smiley Face）

留存率
  100%│▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓
      │                ╲
   50%│                 ╲___
      │                    ╲
    0%└────────────────────────▶ 时间
      Day 0  Day 7  Day 30

特征：早期下降后稳定
含义：产品适应用户需求
代表：Facebook、Slack

2. 持续下滑型（The Hockey Stick）

留存率
  100%│▓▓▓
      │   ╲
   50%│    ╲
      │     ╲
    0%└────────────────────────▶ 时间
      Day 0  Day 7  Day 30

特征：持续下降
含义：产品价值未被感知
代表：大多数失败产品

3. 完美微笑型（The Perfect Smile）

留存率
  100%│▓▓▓▓▓▓▓▓▓▓▓▓▓
      │                    ╱
   50%│                  ╱
      │               ╱
    0%└────────────────────────▶ 时间
      Day 0  Day 7  Day 30

特征：持续增长
含义：网络效应+习惯养成
代表：微信、WhatsApp
```

### Cohort Retention 分析

```
Cohort Retention 表格：

        Week 1   Week 2   Week 3   Week 4   Week 5
Week 1   100%     60%      40%      35%      33%
Week 2   100%     58%      38%      33%      31%
Week 3   100%     55%      36%      32%      -
Week 4   100%     52%      34%      -        -

分析：
- Week 1 → Week 2 流失最多
- 之后留存率趋于稳定
- 关键是优化第一个月的留存
```

## 粘性产品设计

### 每日使用触发

```
每日触发设计：

1. 通知触发
   - 定时通知（早报、晚报）
   - 事件触发（有人@你、有新内容）
   - 社交触发（朋友动态）

2. 习惯触发
   - 时间触发（早上、午休、晚上）
   - 场景触发（通勤、吃饭、睡前）
   - 动作触发（打开某个App后）

3. 奖励触发
   - 每日签到奖励
   - 连续登录奖励
   - 限时奖励

关键：触发要轻柔，不能骚扰
```

### 功能粘性设计

```
功能粘性策略：

1. 核心功能锁定
   - 让核心功能成为不可替代
   - 持续投入核心功能研发
   - 例：Excel的公式功能

2. 数据积累锁定
   - 用户数据越来越多
   - 迁移成本越来越高
   - 例：Notion的笔记、Spotify的歌单

3. 使用习惯锁定
   - 通过重复使用形成习惯
   - 习惯一旦形成很难改变
   - 例：刷抖音的习惯

4. 个性化锁定
   - 产品越来越了解用户
   - 个性化体验无法迁移
   - 例：算法的个性化推荐
```

### 社交粘性设计

```
社交粘性设计：

1. 双向关系
   - 让用户建立真实关系
   - 关系越深，粘性越强
   - 例：微信的聊天记录

2. 群组效应
   - 用户在群组中互动
   - 离开群组=离开关系
   - 例：Slack的工作群

3. 内容积累
   - 用户生成的内容
   - 这些内容让用户无法离开
   - 例：Instagram的照片

4. 社交证明
   - 朋友在这里
   - 社交认同让用户留下
   - 例：LinkedIn的职业社交
```

## 粘性引擎指标

```
粘性核心指标：

1. DAU/MAU（粘性比率）
   - 计算：DAU / MAU × 100%
   - 目标：> 50% 是优秀

2. 核心行为频率
   - 定义：用户完成核心行为的频率
   - 目标：每日或每周

3. 留存率
   - Day 1/7/30 留存
   - 对比同期Cohort

4. 流失率（Churn）
   - 月流失率
   - 目标：< 5%/月

粘性健康检查：
- 留存曲线是否平稳？
- 粘性比率是否提升？
- 用户是否越来越活跃？
```
