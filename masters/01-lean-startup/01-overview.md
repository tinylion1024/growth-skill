# 精益创业体系

> Eric Ries | 不确定环境下的科学增长方法论

---

## 核心理念

精益创业是所有增长方法论的底层操作系统。

### 核心思想

```
┌─────────────────────────────────────────────────────────────┐
│                    不确定环境下的生存法则                      │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│   ❌ 传统方法              ✅ 精益方法                        │
│   ─────────               ─────────                        │
│   完美计划                快速实验                          │
│   大预算                  小MVP                            │
│   直觉判断                数据驱动                         │
│   长周期开发              快速迭代                          │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

**核心原则：**
1. **实验 > 计划** - 市场瞬变，等待完美计划等于死亡
2. **数据 > 直觉** - 用真实用户数据验证假设
3. **快速试错 > 完美执行** - 失败越早，成功越快

## 核心模型

### Build-Measure-Learn 循环

```
        ┌─────────────────────────────────────────────┐
        │                                             │
        ▼                                             │
   ┌─────┐    ┌────────┐    ┌──────┐    ┌────────┐  │
   │ IDEA │───▶│ Build  │───▶│Measure│───▶│ Learn  │  │
   └─────┘    └────────┘    └──────┘    └────────┘  │
      ▲                                              │
      │         快速迭代循环                          │
      └──────────────────────────────────────────────┘
              每次循环都在减少不确定性
```

**关键点：**
- **Build** - 制作最小可行产品（MVP）
- **Measure** - 收集真实用户数据
- **Learn** - 验证或推翻假设，调整方向

### MVP 定义

```
MVP = Minimum Viable Product

┌─────────────────────────────────────────────────────────┐
│                                                         │
│   最少功能         能够验证          不是完美产品       │
│   ─────────       ─────────         ─────────          │
│   只做核心         假设的关键          是最早版本        │
│   功能            用户行为           可以开始迭代       │
│                                                         │
└─────────────────────────────────────────────────────────┘
```

**MVP不是：**
- 不是 Beta 版本
- 不是半成品
- 不是功能齐全的产品

**MVP是：**
- 能验证核心假设的最简单版本
- 足够开始学习循环
- 投入市场获取真实反馈

## 关键概念

### 1. 验证性学习（Validated Learning）

```
验证性学习 vs 虚荣指标

❌ 虚荣指标（Vanity Metrics）
   - 注册用户数
   - 页面浏览量
   - 粉丝数

✅ 验证性指标（Vanity → Actionable）
   - 激活率（完成关键行为）
   - 留存率（持续使用）
   - 转化率（完成目标）
```

### 2. 创新核算（Innovation Accounting）

```
三步验证：

Step 1: 确定基准线
   - 最小产品 MVP
   - 关键指标基准

Step 2: 改善引擎
   - 每次实验改善指标
   - 持续迭代

Step 3: 转向或坚持
   - 数据支持转向 → Pivot
   - 数据支持坚持 → persevere
```

### 3. 虚荣指标 vs 可执行指标

| 类型 | 定义 | 例子 |
|------|------|------|
| 虚荣指标 | 看起来好但无法指导行动 | 总用户数 |
| 可执行指标 | 能指导具体决策 | 激活率、留存率 |

## 实践方法论

### 如何做增长实验

```
实验设计模板：

假设：
   我们相信 [用户群体] 在 [使用场景] 下会 [完成行为]
   因为 [观察/数据]

实验：
   我们将 [改动] 应用在 [目标群体] 上
   预期 [结果变化]

验证：
   如果 [可执行指标] 提升 [X%]
   则假设成立
```

### 实验优先级

```
ICE 框架：

Interest（兴趣）
   - 用户对此有多关心？
   - 1-10分

Confidence（信心）
   - 我对这个想法有多确定？
   - 1-10分

Ease（易度）
   - 实现需要多少资源？
   - 1-10分

ICE = Interest × Confidence × Ease

优先做 ICE 分数最高的实验
```

## 增长应用

### 精益创业 × 增长

```
传统增长：
   Plan → Execute → Result
   （计划赶不上变化）

精益增长：
   Build → Measure → Learn → Iterate
   （持续验证，持续优化）
```

### 常见误区

| 误区 | 说明 | 正确做法 |
|------|------|----------|
| MVP = 功能少 | 容易做出半成品 | MVP要能验证核心假设 |
| 快速 = 粗糙 | 快速迭代不是低质量 | 每次迭代都要有学习 |
| 数据 = 一切 | 忽视定性洞察 | 数据 + 用户访谈结合 |
| 失败 = 结束 | 失败是学习机会 | 快速失败，快速学习 |
