## 完整背景

贝壳找房由链家左晖创立于2018年，前身是链家网，定位为"住"的开放平台。贝壳继承链家20年的行业积累，推出ACN（Agent Cooperation Network）经纪人协作网络，解决传统中介博弈的问题。2020年8月在纽交所上市，成为中国居住服务领域最大的平台。贝壳的核心竞争力是真房源数据库"楼盘字典"和ACN协作网络。

## 增长策略（详细拆解）

### 策略1：ACN经纪人协作网络

**ACN的设计理念**：

```
传统中介的问题：

博弈关系：
  - 房源录入者独享所有佣金
  - 其他经纪人愿意带客户 → 分不到佣金
  - 结果：信息不共享，跳单频发

经纪人博弈：
  - 抢单：谁抢到算谁的
  - 跳单：客户被其他经纪人抢走
  - 互不信任

ACN解决方案：

角色分工：
  - 录入者：录入房源
  - 维护者：维护房源
  - 成交者：成交客户
  - 金融：贷款服务

佣金分配：
  - 按角色分工分配佣金
  - 各环节都有收益
  - 打破独占，激发合作
```

**ACN角色详解**：

```
ACN角色与职责：

1. 房源录入（录入方）
  - 责任：首次录入房源
  - 收益：房源录入奖

2. 房源维护（维护方）
  - 责任：日常维护房源
  - 收益：房源维护费

3. 客源成交（成交方）
  - 责任：成交客户
  - 收益：主要成交佣金

4. 金融服务（金融方）
  - 责任：按客贷款服务
  - 收益：金融服务费

5. 交易管家（管家方）
  - 责任：签约后服务
  - 收益：交易服务费

6. 客源开发（开发方）
  - 责任：开发客户
  - 收益：客源开发费

多方协作：
  - 各司其职
  - 利益共享
  - 打破博弈
```

**ACN的价值**：

```
ACN解决的问题：

1. 房源共享：
  - A录入的房源 → B也可以卖
  - 房源充分曝光
  - 成交效率提升

2. 客源共享：
  - A开发的客户 → 可以由B成交
  - 经纪人合作
  - 客户体验好

3. 跳单问题：
  - 平台记录所有贡献者
  - 佣金按规则分配
  - 跳单无意义

4. 经纪人效率：
  - 专注自己擅长的环节
  - 整体效率提升
  - 收入稳定
```
