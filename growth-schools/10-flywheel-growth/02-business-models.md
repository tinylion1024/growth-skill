# 增长飞轮 - 商业模型

> David Skok | 增长必须能赚钱

---

## 核心理念

```
增长 ≠ 赚钱

传统思维：增长优先        商业思维：增长必须健康
用户增长 = 成功          用户增长 + 盈利 = 成功
规模越大越好             规模要匹配商业模式
后期考虑盈利              盈利是增长的前提

核心原则：
1. LTV > CAC - 用户必须赚回成本
2. 留存决定命运 - 没有留存，增长是空谈
3. 指标驱动决策 - 数据验证商业模型
```

---

## 核心指标体系

```
SaaS 关键指标：

用户指标：
- MAU（月活跃用户）
- 激活率
- 留存率
- 流失率

财务指标：
- ARR（年度经常性收入）
- MRR（月经常性收入）
- ARPU（每用户收入）
- LTV（用户终身价值）

效率指标：
- CAC（获客成本）
- LTV/CAC（比率）
- Payback Period（回收周期）
```

## LTV/CAC 比率

```
LTV/CAC 比率图：

5+  ━━━━━━━━━━━━━━━━━━━━━━━━━━  大力扩张
3   ━━━━━━━━━━━━━━━━━━━━━━━━━━━━  安全边界
1   ━━━━━━━━━━━━━━━━━━━━━━━━━━━━  盈亏平衡
0   ━━━━━━━━━━━━━━━━━━━━━━━━━━━━  亏损

不同阶段目标：
- 早期：LTV/CAC > 1（先活下来）
- 成长：LTV/CAC > 3（健康增长）
- 成熟：LTV/CAC > 5（高效扩张）

如果LTV/CAC < 3：
- 优化获客效率
- 提高客单价
- 延长客户生命周期
```

## 流失率（Churn）

```
流失率类型：

1. 客户流失（Customer Churn）
   计算：流失客户数 / 期初客户数

2. 收入流失（Revenue Churn）
   Gross Revenue Churn：流失的收入
   Net Revenue Churn：流失 - 扩张

流失率基准（按月）：
- < 1%/月：优秀
- 1-2%/月：良好
- 2-5%/月：一般
- > 5%/月：危险
```

## 单位经济验证

```
基础公式：

LTV = ARPU × 留存月数 × 毛利率

验证标准：
- LTV/CAC > 3
- Payback Period < 12个月
- 流失率 < 5%/月

如果不达标，问题在哪？

问题        可能原因           解决方案
────────────────────────────────────────
LTV太低    定价太低           提高定价
           用户生命周期短       提升留存

CAC太高    渠道效率低         优化渠道
           转化率低            优化漏斗

流失高     产品价值不足        提升产品
           用户匹配不对        优化获客
```

## 精益分析四步法

```
Step 1: 找到那个关键指标（One Metric That Matters）
   - 现阶段最重要的指标是什么？

Step 2: 建立基准线
   - 当前指标值是多少？
   - 与行业比如何？

Step 3: 设定目标
   - 目标值是多少？
   - 什么时候达成？

Step 4: 设计实验
   - 什么实验能改善这个指标？
   - 优先级是什么？
```
