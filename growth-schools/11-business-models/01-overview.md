## LTV（用户终身价值）

```
LTV 定义与计算：

LTV = ARPU × 毛利率 × 平均生命周期（月份）

LTV 计算示例：
- 月ARPU = $100
- 月流失率 = 5%
- 毛利率 = 70%

平均生命周期 = 1 / 流失率 = 1 / 0.05 = 20个月

LTV = $100 × 20 × 70% = $1,400
```

---

## CAC（用户获取成本）

```
CAC 定义与计算：

CAC（简单）= 营销总支出 / 新增客户数

CAC（完整）= (营销支出 + 销售支出 + 相关人员成本)
            ─────────────────────────────────────
                      新增付费客户数

CAC 优化方向：
1. 渠道效率 - 找到更低CAC的渠道
2. 转化率 - 优化从线索到客户的转化
3. 单位成本 - 谈判更好的媒体价格
```

## Payback Period（回收周期）

```
回收周期 = CAC / (ARPU × 毛利率)

回收周期参考：
< 6个月   优秀   - 资金效率高，快速扩张
6-12个月  良好   - 可接受，持续投入
12-18个月 一般   - 需要优化，降低风险
> 18个月  危险   - 资金压力大，考虑转型

SaaS基准：< 12个月是安全线
```

---

## Churn（流失率）

```
流失率类型：

1. 客户流失（Customer Churn）
   计算：流失客户数 / 期初客户数
   目标：< 5%/年（大型企业SaaS）

2. 收入流失（Revenue Churn）
   - Gross Revenue Churn：流失的收入
   - Net Revenue Churn：流失 - 扩张
   - NDR > 100%：扩张收入 > 流失收入

流失率基准（月）：
- < 1%/月：优秀
- 1-2%/月：良好
- 2-5%/月：一般
- > 5%/月：危险
```

## 增长健康度判断

```
增长健康度矩阵：

                    LTV/CAC > 3        LTV/CAC < 3
                    ─────────────      ─────────────
  流失率 < 5%/月    ██ 健康发展        ██ 优化效率
  月流失率

  流失率 > 10%/月   ██ 加速获客        ██ 危险信号

建议：
- LTV/CAC > 3 + 流失 < 5%：加速增长
- LTV/CAC < 3 + 流失 < 5%：优化获客效率
- LTV/CAC > 3 + 流失 > 10%：专注留存
- LTV/CAC < 3 + 流失 > 10%：停止增长，先解决问题
```

## 问题诊断矩阵

```
指标问题 → 原因分析 → 解决方案

问题：LTV/CAC < 3
├─ CAC太高
│   ├─ 渠道效率低 → 优化渠道组合
│   ├─ 转化率低 → 优化漏斗
│   └─ 单位成本高 → 谈判/优化
│
└─ LTV太低
    ├─ 定价低 → 涨价
    ├─ 生命周期短 → 提升留存
    └─ 毛利率低 → 优化成本结构

问题：Payback Period > 18个月
├─ CAC太高 → 降低获客成本
└─ ARPU太低 → 提高客单价

问题：流失率 > 5%/月
├─ 产品价值不足 → 提升产品
├─ 用户匹配不对 → 优化获客
└─ 竞争流失 → 差异化
```
