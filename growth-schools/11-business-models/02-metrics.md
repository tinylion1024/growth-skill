# 商业模型 - 核心指标详解

> LTV / CAC / Churn / ARR | SaaS增长健康度

---

## 核心指标体系

### LTV（用户终身价值）

```
LTV 定义与计算：

┌─────────────────────────────────────────────────────────────────┐
│                                                                 │
│   LTV = ARPU × 毛利率 × 平均生命周期（月份）                    │
│                                                                 │
│   或者：                                                        │
│                                                                 │
│   LTV = 月ARPU × 留存月数 × 毛利率                              │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘

LTV 计算示例：

假设：
- 月ARPU = $100
- 月流失率 = 5%
- 毛利率 = 70%

平均生命周期 = 1 / 流失率 = 1 / 0.05 = 20个月

LTV = $100 × 20 × 70% = $1,400
```

### CAC（用户获取成本）

```
CAC 定义与计算：

┌─────────────────────────────────────────────────────────────────┐
│                                                                 │
│   CAC（简单）= 营销总支出 / 新增客户数                           │
│                                                                 │
│   CAC（完整）= (营销支出 + 销售支出 + 相关人员成本)              │
│                ───────────────────────────────────────          │
│                          新增付费客户数                          │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘

CAC 优化方向：

1. 渠道效率
   - 找到更低CAC的渠道
   - 优化渠道ROI

2. 转化率
   - 优化从线索到客户的转化
   - 降低客户获取摩擦

3. 单位成本
   - 广告效率优化
   - 谈判更好的媒体价格
```

### LTV/CAC 比率

```
LTV/CAC 比率详解：

┌─────────────────────────────────────────────────────────────────┐
│                                                                 │
│                    LTV/CAC 比率图                               │
│                                                                 │
│   5+  ━━━━━━━━━━━━━━━━━━━━━━━━━━  大力扩张                      │
│                                                                 │
│   3   ━━━━━━━━━━━━━━━━━━━━━━━━━━━━  安全边界                    │
│                                                                 │
│   1   ━━━━━━━━━━━━━━━━━━━━━━━━━━━━  盈亏平衡                    │
│                                                                 │
│   0   ━━━━━━━━━━━━━━━━━━━━━━━━━━━━  亏损                          │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘

不同阶段的目标：

早期阶段：LTV/CAC > 1（先活下来）
成长阶段：LTV/CAC > 3（健康增长）
成熟阶段：LTV/CAC > 5（高效扩张）

如果LTV/CAC < 3：
- 优化获客效率
- 提高客单价
- 延长客户生命周期
- 降低服务成本
```

### Payback Period（回收周期）

```
回收周期定义：

┌─────────────────────────────────────────────────────────────────┐
│                                                                 │
│   Payback Period = CAC / (ARPU × 毛利率)                       │
│                                                                 │
│   公式解读：                                                    │
│   - CAC：获取一个客户的成本                                      │
│   - ARPU × 毛利率：每个客户每月贡献的毛利                        │
│   - 回收周期：需要多少个月才能收回成本                           │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘

回收周期参考：

< 6个月   优秀   - 资金效率高，快速扩张
6-12个月  良好   - 可接受，持续投入
12-18个月 一般   - 需要优化，降低风险
> 18个月  危险   - 资金压力大，考虑转型

SaaS基准：< 12个月是安全线
```

### Churn（流失率）

```
流失率类型：

1. 客户流失（Customer Churn）
   - 计算：流失客户数 / 期初客户数
   - 目标：< 5%/年（大型企业SaaS）

2. 收入流失（Revenue Churn）
   - 计算：流失MRR / 期初MRR
   - Gross Revenue Churn：流失的收入
   - Net Revenue Churn：流失 - 扩张（扩张>流失=负流失）

3. 净推荐值流失（Net Dollar Retention）
   - 定义：现有客户在12个月内的贡献收入变化
   - NDR > 100%：扩张收入 > 流失收入

流失率基准（月）：

- < 1%/月：优秀
- 1-2%/月：良好
- 2-5%/月：一般
- > 5%/月：危险

年流失率：
- < 10%/年：优秀
- 10-20%/年：良好
- 20-30%/年：一般
- > 30%/年：危险
```

### ARR / MRR

```
ARR（年度经常性收入）：

┌─────────────────────────────────────────────────────────────────┐
│                                                                 │
│   ARR = MRR × 12                                               │
│                                                                 │
│   或者：                                                        │
│                                                                 │
│   ARR = 年度合同总价值                                          │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘

ARR 增长驱动：

New ARR（新增ARR）
  - 新客户
  - 新增合同

Expansion ARR（扩张ARR）
  - 现有客户扩购
  - 升级订阅

Churned ARR（流失ARR）
  - 客户流失
  - 降级订阅

Net New ARR = New ARR + Expansion ARR - Churned ARR
```

## 指标诊断框架

### 增长漏斗诊断

```
增长漏斗指标：

┌─────────────────────────────────────────────────────────────────┐
│                                                                 │
│   认知 (Awareness)                                             │
│       │                                                        │
│       ▼                                                        │
│   流量 (Traffic)        ─ 量化：访客数                         │
│       │                                                        │
│       ▼                                                        │
│   转化 (Conversion)     ─ 量化：转化率                         │
│       │                                                        │
│       ▼                                                        │
│   激活 (Activation)     ─ 量化：激活率                         │
│       │                                                        │
│       ▼                                                        │
│   留存 (Retention)      ─ 量化：留存率                          │
│       │                                                        │
│       ▼                                                        │
│   变现 (Revenue)        ─ 量化：ARPU                           │
│       │                                                        │
│       ▼                                                        │
│   推荐 (Referral)        ─ 量化：K-factor                       │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘

漏斗优化：
- 每个环节都有优化空间
- 从上到下优化效率递减
- 聚焦阻力最大的环节
```

### 问题诊断矩阵

```
指标问题 → 原因分析 → 解决方案

问题：LTV/CAC < 3
├─ CAC太高
│   ├─ 渠道效率低 → 优化渠道组合
│   ├─ 转化率低 → 优化漏斗
│   └─ 单位成本高 → 谈判/优化
│
└─ LTV太低
    ├─ 定价低 → 涨价
    ├─ 生命周期短 → 提升留存
    └─ 毛利率低 → 优化成本结构

问题：Payback Period > 18个月
├─ CAC太高 → 降低获客成本
└─ ARPU太低 → 提高客单价

问题：流失率 > 5%/月
├─ 产品价值不足 → 提升产品
├─ 用户匹配不对 → 优化获客
└─ 竞争流失 → 差异化
```
