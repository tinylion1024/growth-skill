```
好产品自己会增长

传统思维：              产品驱动思维：
增长 = 拉新            增长 = 产品价值
销售驱动               产品驱动
```

## 用户激活框架

```
AARRR → HEART

传统 AARRR：           Google HEART:
Acquisition → 获取      Happiness → 用户愉悦度
Activation → 激活 ←关键  Engagement → 用户参与度
Retention → 留存        Adoption → 产品采用率
Revenue → 变现          Retention → 用户留存
Referral → 推荐          Task Success → 任务成功率
```

## Freemium 设计原则

```
免费层                   付费层
┌─────────┐             ┌─────────┐
│  核心   │  ───────▶   │  高级   │
│  功能   │             │  功能   │
└─────────┘             └─────────┘
    │                       │
    │    提供足够价值        │
    │    让用户体验到核心价值  │
    ▼                       ▼

关键：免费功能要能解决问题，付费功能要有明显差异
```

## 留存曲线分析

```
留存曲线类型：

1. 快乐台车型（Facebook、Slack）
   早期下降后稳定

2. 持续下滑型
   需要持续改善

3. 完美微笑型（微信、WhatsApp）
   持续增长（网络效应）

关键指标：Day 30 留存 > 20%（好），>10%（一般）
```

## Onboarding 七原则

| 原则 | 说明 |
|------|------|
| 减少摩擦 | 步骤最少，默认值合理 |
| 展示价值 | 早期展示核心价值 |
| 及时反馈 | 每步操作都有反馈 |
| 社交证明 | 显示"已有X人使用" |
| 里程碑设计 | 完成关键行为后庆祝 |
| 个性化 | 不同用户不同路径 |
| 持续优化 | A/B测试持续改善 |

## Aha Moment 框架

```
定义：用户第一次真正体验到产品价值的时刻

找到 Aha Moment：
- 分析数据：用户做什么行为后开始留存？
- 用户访谈：用户说"这个产品真有用"是什么时候？
- 分群对比：留存用户 vs 流失用户的行为差异

优化路径：
新用户 → 发现价值 → 体验价值 → 建立习惯
    │           │           │
 快速找到      感受到       持续回来
 核心功能      核心价值      使用产品
```
