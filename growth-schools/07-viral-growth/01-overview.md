# 病毒传播增长

> Jonah Berger | 让用户主动传播

---

## 核心理念

```
病毒传播 = 让用户主动分享 + 邀请新用户

核心公式：
Viral Growth = f(Social Currency, Emotion, Trigger, Public, Practical Value, Story)

核心原则：
1. 传播是用户行为，不是平台push
2. 内容/产品要有分享动机
3. 机制设计让传播更容易
4. K-Factor > 1才能病毒
```

---

## 病毒传播模型

### K-Factor详解

```
K-Factor（病毒系数）：

K = i × c

i = 每个用户发送的邀请数
c = 每个邀请的转化率

判断标准：
- K > 1：病毒式增长
- K < 1：衰减式增长

优化方向：
1. 提高邀请数（i）
   - 找到好的触发点
   - 降低邀请摩擦
   - 激励分享

2. 提高转化率（c）
   - 优化邀请内容
   - 优化落地页
   - 减少注册步骤
```

### 病毒传播类型

```
病毒传播分类：

1. 原生病毒（Inherent Viral）
   - 用户为了使用产品而分享
   - 例：Hotmail底部署名
   - 效果最好，成本最低

2. 激励病毒（Incentivized Viral）
   - 用户因奖励而分享
   - 例：Dropbox邀请送空间
   - 效果好，成本可控

3. 口碑病毒（Word of Mouth）
   - 用户因喜欢而推荐
   - 例：好的产品体验
   - 效果慢，长期价值大

4. 场景病毒（Contextual Viral）
   - 用户在特定场景分享
   - 例：抖音视频分享
   - 需要持续产出内容
```

## STEPPS病毒传播框架

```
Jonah Berger 的 STEPPS 框架：

S - Social Currency（社交货币）
   人们分享让自己显得酷的东西

   案例：微信红包（分享显得大方）

T - Triggers（触发器）
   想到你就会提到你

   案例：星巴克（喝咖啡就会想到）

E - Emotion（情感）
   引起情绪反应的内容会传播

   案例：感人故事、有趣视频

P - Public（公共性）
   能被看到的东西更容易传播

   案例：排队、爆款

P - Practical Value（实用价值）
   有用的信息会传播

   案例：省钱技巧、教程

S - Story（故事）
   人们喜欢讲故事

   案例：品牌故事、产品故事
```

## 病毒传播设计

### 分享机制设计

```
分享机制设计要素：

触发点设计：
- 使用后分享
- 达到里程碑分享
- 遇到问题时分享

分享内容设计：
- 视觉化（截图、照片）
- 可定制（个性化）
- 一键分享

激励设计：
- 邀请人奖励
- 被邀请人奖励
- 排行榜竞争

案例：拼多多
- 触发点：低价商品、砍价
- 分享内容：价格对比、朋友ID
- 激励：免费获得商品
- 结果：病毒式增长
```

### 裂变增长模型

```
裂变增长模型：

┌─────────────────────────────────────────────────────────────────┐
│                                                                 │
│   用户A ──邀请──▶ 用户B ──邀请──▶ 用户C                        │
│      │                   │                                      │
│      │  分享链接          │  分享链接                           │
│      │  触发分享          │  触发分享                           │
│      │                   │                                      │
│      │  转化率c           │  转化率c                            │
│      │                   │                                      │
│      ▼                   ▼                                      │
│   新用户              新用户                                    │
│                                                                 │
│   K = i × c = 0.5 × 0.5 = 0.25                                 │
│   每个用户带来0.25个新用户                                      │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘

裂变效率优化：
- 降低分享门槛
- 提高分享触发频率
- 优化被邀请人体验
- 加速循环时间
```

## 经典案例

### Dropbox邀请机制

```
Dropbox病毒增长：

机制：
- 邀请1位好友，双方获得500MB额外空间
- 最多获得16GB免费空间

效果：
- K-Factor = 0.35（每个用户邀请0.35个新用户）
- 虽然K < 1，但边际成本 ≈ 0
- 贡献了35%的增长

关键洞察：
- 不是K > 1才算成功
- 关键是成本要低
- 持续的小规模病毒也很好
```

### 拼多多砍价

```
拼多多病毒增长：

机制：
- 邀请朋友帮砍价
- 砍到0元可以免费拿
- 需要30-50人才能砍到0元

效果：
- K-Factor > 1
- 几乎0成本获客
- 成为增长主要引擎

关键洞察：
- 利益驱动是强触发
- 社交关系是背书
- 越砍越想砍（损失厌恶）
```

## 病毒传播指标

```
病毒传播核心指标：

1. K-Factor（病毒系数）
   - K = i × c
   - 目标：K > 1

2. 病毒周期（Viral Cycle Time）
   - 从邀请到转化的时间
   - 越短越好

3. 病毒贡献率
   - 病毒获取用户 / 总新增用户
   - 越高越好

4. 邀请转化率
   - 收到邀请的人注册比例
   - 优化落地页和邀请内容
```

## 常见误区

```
病毒传播误区：

❌ 只追求K > 1
   - K < 1也可以有价值
   - 关键是成本和效率

❌ 忽视产品质量
   - 病毒只是放大器
   - 产品不好，病毒会反噬

❌ 过度依赖病毒
   - 病毒不稳定
   - 需要其他增长方式配合

❌ 强行病毒
   - 用户不是傻子
   - 强制分享会伤害体验

✅ 正确做法：
- 产品价值是基础
- 让分享变得自然
- 激励要合理
- 配合其他增长方式
```
