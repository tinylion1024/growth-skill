# 增长黑客派 - 精益创业理论

> Eric Ries | 不确定环境下的科学增长方法论

---

## 核心理念

```
精益创业 = 底层操作系统

核心公式：
Growth = f(Experiment, Data, Speed)

核心原则：
1. 实验 > 计划
2. 数据 > 直觉
3. 快速试错 > 完美执行
```

## 核心模型

### MVP 定义

```
MVP = Minimum Viable Product

┌─────────────────────────────────────────────────────────┐
│  最少功能    │    能够验证     │    不是完美产品        │
│  只做核心    │    假设关键      │    最早版本            │
└─────────────────────────────────────────────────────────┘

MVP不是：
- 不是Beta版本
- 不是半成品

MVP是：
- 能验证核心假设的最简单版本
- 足够开始学习循环
```

---

## 验证性学习

```
验证性学习 vs 虚荣指标

❌ 虚荣指标：注册用户数、页面浏览量、粉丝数

✅ 可执行指标：激活率、留存率、转化率
```

---

## 创新核算三步法

```
Step 1: 确定基准线
   - MVP上线后，测量关键指标作为基准

Step 2: 改善引擎
   - 通过实验改善核心指标

Step 3: 决策：坚持 or 转向
   - 数据支持继续 → 坚持
   - 数据支持改变 → 转向（Pivot）
```

---

## 五种转向（Pivot）

| 类型 | 说明 |
|------|------|
| 放大转向 | 从一个功能变成整个产品 |
| 缩小转向 | 从一个产品变成更大产品的一部分 |
| 客户细分转向 | 换一批目标用户 |
| 渠道转向 | 通过不同渠道触达同样用户 |
| 技术转向 | 用不同技术达到同样效果 |

---

## ICE 优先级框架

```
ICE = Interest × Confidence / Ease

Interest（兴趣）
   - 用户对此有多关心？1-10分

Confidence（信心）
   - 我对这个想法有多确定？1-10分

Ease（易度）
   - 实现需要多少资源？1-10分

优先做 ICE 分数最高的实验
```

## 常见误区

| 误区 | 正确做法 |
|------|----------|
| MVP = 功能少 | MVP要能验证核心假设 |
| 快速 = 粗糙 | 每次迭代都要有学习 |
| 数据 = 一切 | 数据 + 用户访谈结合 |
| 失败 = 结束 | 快速失败，快速学习 |
